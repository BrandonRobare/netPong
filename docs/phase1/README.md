# netPong Phase 1

Tag: `v0.2-phase1-analysis`

Phase 1 is the design and analysis milestone for turning the baseline Pong assignment into `netPong`.

## What is in this milestone

- `Phase1_Report.md` — architecture analysis and RFC-driven design notes

## What changed

This milestone does not change runtime behavior. I kept the baseline source the same and wrote down how the game logic has to change to support left/right court views, socket handoff, miss handling, and timer ownership.
